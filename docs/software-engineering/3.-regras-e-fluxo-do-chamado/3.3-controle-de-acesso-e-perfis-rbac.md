# 3.3 Controle de Acesso e Perfis (RBAC)

#### 3.3.1 Matriz de Permissões e Níveis de Acesso

A tabela abaixo detalha as capacidades de cada perfil dentro do sistema:

| **`admin`**     | Administrador geral e gestor principal da organização.               | Gerenciar (criar, editar, excluir) dados da empresa, alterar o cargo de outros usuários e atuar na resolução de chamados.             | **Acesso Total:** Telas de Empresa, Lojas/Departamentos, Categorias, Usuários e Chamados.                                             |
| --------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **`attendant`** | Profissional da equipe de suporte técnico e atendimento.             | Interagir, responder, atualizar status e fechar chamados.                                                                             | **Acesso Operacional:** Tela de Chamados (Acesso total). Visualiza as telas de Lojas, Categorias e Empresa apenas no modo de leitura. |
| **`employee`**  | Funcionário padrão (solicitante de suporte).                         | Abrir, cancelar, fechar chamados e interagir  exclusivamente nos chamados que ele mesmo criou.                                        | **Acesso Restrito:** Apenas a tela de Chamados (além das telas de acesso universal).                                                  |
| **`iddle`**     | Usuário recém-cadastrado ou ocioso, sem vínculo com nenhuma empresa. | Criar uma nova empresa no sistema (sendo promovido a `admin`) ou enviar uma solicitação/convite para entrar em uma empresa existente. | **Sem Acesso Operacional:** Apenas as Telas de Acesso Universal (Início de decisão, Perfil, Configurações, Convites).                 |

#### 3.3.2 Telas de Acesso Universal

Independentemente do cargo ou do vínculo empresarial, o sistema garante a todos os usuários autenticados o acesso às seguintes rotas estruturais:

* **Início (Dashboard):** Tela principal, cujo conteúdo se adapta ao perfil do usuário.
* **Perfil do Usuário:** Gestão de dados pessoais e de conta.
* **Configurações:** Preferências do aplicativo.
* **Convites:** Caixa de entrada para gerenciar solicitações de entrada em organizações.

Todos os usuários podem sair de suas empresas.

#### 3.3.3 O Fluxo de Onboarding (Cargo "Iddle")

Para evitar inconsistências no banco de dados (como um usuário tentando abrir um chamado sem pertencer a uma empresa), todo novo cadastro recebe o cargo de `iddle`.

Na tela de Início, o usuário `iddle` é bloqueado no dashboard padrão e recebe apenas duas opções de transição:

* **Participar de uma Empresa:** Envia uma solicitação para uma organização existente. Ao ser aprovado pelo `admin`, assume o cargo designado e as telas são desbloqueadas.
* **Criar uma Empresa:** Inicia o processo de cadastro de uma nova organização. Ao finalizar, o sistema o promove automaticamente a `admin`, vinculando-o à nova base e liberando o gerenciamento.
