# 1. Engenharia de Requisitos

### 2.1 Requisitos Funcionais (RF):

Os requisitos funcionais definem o que o sistema deve fazer.

| ID   | Requisito            | Descrição                                                                                                                                   |
| ---- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| RF01 | Gestão de Unidades   | O sistema deve permitir o cadastro de empresas e suas múltiplas lojas.                                                                      |
| RF02 | Perfil de Usuário    | O sistema deve diferenciar acessos entre "Atendente" (Suporte) e "Funcionário" (Solicitante).                                               |
| RF03 | Abertura de Chamados | O funcionário deve poder abrir chamados e através de um formulário preencher as informações básicas (título, descrição, anexos, prioridade) |
| RF04 | Chat Interno         | Cada chamado deve possuir um chat em tempo real para comunicação entre as partes.                                                           |
| RF05 | Gestão de Mídia      | O chat deve suportar o envio de fotos, documentos PDF e mensagens de áudio.                                                                 |
| RF06 | Controle de Status   | O sistema deve permitir transições de status: Aberto, Em Atendimento, Pendente e Finalizado.                                                |
| RF07 | Histórico de Lojas   | O sistema deve registrar a qual loja o chamado pertence, mesmo que o funcionário mude de unidade depois.                                    |
| RF08 | Data do Chamado      | O sistema deve armazenar a data em que o chamado iniciou e finalizou. E mudança de status também.                                           |
