# 2.1 Requisitos Funcionais

### 2.1 Requisitos Funcionais (RF)

Os requisitos funcionais definem o que o sistema deve fazer.

#### 2.1.1 Módulo de Cadastro e Acesso

| ID   | Requisito          | Descrição                                                                                     |
| ---- | ------------------ | --------------------------------------------------------------------------------------------- |
| RF01 | Gestão de Unidades | O sistema deve permitir o cadastro de empresas e seus funcionários.                           |
| RF02 | Perfil de Usuário  | O sistema deve diferenciar acessos entre "Atendente" (Suporte) e "Funcionário" (Solicitante). |

#### 2.1.2 Módulo de Chamados

| ID   | Requisito            | Descrição                                                                                                                                       |
| ---- | -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| RF03 | Abertura de Chamados | O funcionário deve poder abrir chamados e, através de um formulário, preencher as informações básicas (título, descrição, anexos e prioridade). |
| RF07 | Histórico de Lojas   | O sistema deve registrar a qual loja o chamado pertence, mesmo que o funcionário mude de unidade depois.                                        |
| RF08 | Data do Chamado      | O sistema deve armazenar a data em que o chamado iniciou, finalizou e teve mudança de status.                                                   |

#### 2.1.3 Módulo de Atendimento e Comunicação

| ID   | Requisito          | Descrição                                                                                                                                                                                                                    |
| ---- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF04 | Chat Interno       | Cada chamado deve possuir um chat em tempo real para comunicação entre as partes.                                                                                                                                            |
| RF05 | Gestão de Mídia    | O chat deve suportar o envio de fotos e documentos PDF.                                                                                                                                                                      |
| RF06 | Controle de Status | O sistema deve permitir transições de status: Aberto, Em Atendimento, Pendente, Resolvido, Finalizado e Cancelado, conforme as regras definidas em [3.2 Status do Chamado](../3.-regras-de-negocio/3.2-fluxo-de-chamado.md). |

#### 2.1.4 Módulo de Visualização e Acompanhamento

| ID   | Requisito             | Descrição                                                                                                       |
| ---- | --------------------- | --------------------------------------------------------------------------------------------------------------- |
| RF09 | Painel do Funcionário | O funcionário deve ter uma visão filtrada contendo apenas os chamados que ele mesmo abriu.                      |
| RF10 | Panorama do Atendente | O atendente deve visualizar um panorama global de todos os chamados abertos por ele mesmo e chamados pendentes. |

#### 2.1.5 Módulo de Administração de Usuários

| ID   | Requisito          | Descrição                                                                                             |
| ---- | ------------------ | ----------------------------------------------------------------------------------------------------- |
| RF11 | Gestão de Usuários | O administrador da empresa deve ser capaz de cadastrar, editar e desativar Funcionários e Atendentes. |
