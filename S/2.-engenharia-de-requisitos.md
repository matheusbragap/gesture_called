# 2. Engenharia de Requisitos

### 2.1 Requisitos Funcionais (RF):

Os requisitos funcionais definem o que o sistema deve fazer.

| ID   | Requisito            | Descrição                                                                                                                                   |
| ---- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| RF01 | Gestão de Unidades   | O sistema deve permitir o cadastro de empresas e suas múltiplas lojas.                                                                      |
| RF02 | Perfil de Usuário    | O sistema deve diferenciar acessos entre "Atendente" (Suporte) e "Funcionário" (Solicitante).                                               |
| RF03 | Abertura de Chamados | O funcionário deve poder abrir chamados e através de um formulário preencher as informações básicas (título, descrição, anexos, prioridade) |
| RF04 | Chat Interno         | Cada chamado deve possuir um chat em tempo real para comunicação entre as partes.                                                           |
| RF05 | Gestão de Mídia      | O chat deve suportar o envio de fotos, documentos PDF e mensagens de áudio.                                                                 |
| RF06 | Controle de Status   | O sistema deve permitir transições de status: Aberto, Em Atendimento, Pendente e Finalizado.                                                |
| RF07 | Histórico de Lojas   | O sistema deve registrar a qual loja o chamado pertence, mesmo que o funcionário mude de unidade depois.                                    |
| RF08 | Data do Chamado      | O sistema deve armazenar a data em que o chamado iniciou e finalizou. E mudança de status também.                                           |

### 2.2 Requisitos Não Funcionais (RNF)

Os requisitos não funcionais definem os critérios técnicos e de qualidade do sistema.

| ID    | Categoria          | Descrição                                                                                                                                      |
| ----- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| RNF01 | Portabilidade      | O sistema deve ser multiplataforma, desenvolvido em Flutter, com funcionamento em ambiente Web e em dispositivos Android.                      |
| RNF02 | Tecnologia Backend | A API deve ser desenvolvida em Node.js, seguindo uma arquitetura RESTful para a comunicação com o frontend.                                    |
| RNF03 | Segurança          | Toda comunicação entre aplicação e servidor deve ser autenticada por token JWT, garantindo acesso apenas a usuários autenticados.              |
| RNF04 | Usabilidade        | A interface deve ser responsiva, adaptando-se automaticamente a diferentes tamanhos de tela, como celulares e monitores.                       |
| RNF05 | Armazenamento      | O sistema deve gerenciar o upload de mídias, como áudios, imagens e PDFs, de forma eficiente, sem comprometer a performance do banco de dados. |
| RNF06 | Persistência       | Os dados devem ser armazenados em banco de dados relacional, garantindo a integridade das relações entre lojas e funcionários.                 |
| RNF07 | Disponibilidade    | O sistema deve ser projetado para suportar acessos simultâneos de múltiplos usuários, em diferentes lojas da mesma empresa.                    |
