# 3. Regras de Negócio

### 3.1 Regras de Negócio (RN)

As regras de negócio definem restrições operacionais e comportamentos obrigatórios do sistema.

| ID   | Regra                  | Descrição                                                                                                                                                                                                                     |
| ---- | ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RN01 | Vínculo de Unidade     | Ao abrir um chamado, o sistema deve vinculá-lo à loja atual do funcionário. Se houver transferência de loja posteriormente, o chamado deve permanecer associado à loja de origem para fins de histórico e apuração de custos. |
| RN02 | Hierarquia de Acesso   | Funcionários devem visualizar apenas os chamados abertos por eles mesmos. Atendentes devem visualizar todos os chamados da empresa, independentemente da loja de origem.                                                      |
| RN03 | Finalização do Chamado | Um chamado só pode ser alterado para o status **Finalizado** quando houver pelo menos uma interação no chat confirmando a resolução do problema.                                                                              |
| RN04 | Limite de Anexos       | O sistema deve aceitar anexos nos formatos de áudio, imagem e PDF, com limite máximo de **10 MB por arquivo**, a fim de evitar sobrecarga no backend.                                                                         |
